## 👋 What is this?

If you're new to programming, setting up a web development environment on your laptop can be tricky — browsers, dev tools, file paths, "where do I put my files?", etc.

This repository skips all of that. When you click the button:

1. GitHub creates a **complete coding environment** for you in the cloud (free).
2. Visual Studio Code opens **in your browser**, with HTML/CSS/JS ready to go.
3. You write code, hit ▶ Show Preview, and see your webpage live — beside the code.

No downloads. No installation. Works on **any** computer with a browser, including Chromebooks and iPads.

## 📋 Step-by-Step (First Time)

1. **Click the green button above.**
2. If asked, **sign in to GitHub** with your free account.
   - Don't have one? It takes ~30 seconds at [github.com/signup](https://github.com/signup).
3. Wait about 15 seconds while GitHub opens your environment.
4. The VS Code editor will appear in your browser. **You're in!**
5. The file `index.html` will already be open. Click the ▶ **Show Preview** icon in the top-right corner of the editor.
6. Your webpage appears in a panel beside the code. Try editing the heading in `index.html` and saving — the preview updates automatically. 🎉

## 💡 Tips

- **Your work is saved automatically** while the Codespace is open.
- Codespaces sleep after 30 minutes of inactivity (your code is kept; just click the button again to resume).
- GitHub gives every free account **60 free hours per month** of Codespaces — plenty for this course.
- The three files in your project work together:
  - `index.html` — the **content** (text, buttons, images)
  - `styles.css` — how it **looks** (colours, fonts, layout)
  - `script.js` — how it **behaves** (clicks, animations, interactions)
- To create a new file: right-click in the sidebar → New File → name it `something.html` (or `.css`, or `.js`).
- **To save the whole project to your computer:** click the 📦 **Download my work** button at the bottom-left of the screen, then right-click the new `.zip` file that appears in the Explorer → **Download**.
- **To pick up where you left off later:** drag your saved `my-work-*.zip` from your computer onto the Explorer panel (left sidebar), then click the 📂 **Restore my work** button at the bottom-left.

## ❓ Troubleshooting

| Problem | Fix |
|---|---|
| Button does nothing | Make sure you're signed in to GitHub first |
| "You've reached your Codespaces limit" | Delete old Codespaces at [github.com/codespaces](https://github.com/codespaces) |
| Preview doesn't appear | Make sure you're viewing an `.html` file, then click the ▶ Show Preview icon in the top-right |
| Preview doesn't update when I save | Make sure your file is saved (look for the dot ● on the file tab — if it's there, the file is unsaved). Press Ctrl+S to save. |
| Browser asks to "See text and images copied to the clipboard" | Click **Allow** — it's a one-time browser permission so paste works in the editor |
| "No zip file found" when I click 📂 Restore my work | Drag the `.zip` from your computer's Downloads folder onto the Explorer panel first, wait for it to upload, then click Restore again |
| Need to start fresh | Close the tab, delete the Codespace, click the button again |

If you're stuck, ask in class or post in the module forum.
